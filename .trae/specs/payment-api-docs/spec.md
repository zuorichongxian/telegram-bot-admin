# 支付接口文档展示 Spec

## Why

当前"拉单测试1"页面虽然实现了支付接口的调用功能，但缺少完整的接口文档说明。用户需要能够查看每个接口的详细参数信息，包括输入参数、输出参数、是否必填、参数类型和描述等，以便更好地理解和使用这些接口。

## What Changes

- 在"拉单测试1"页面添加接口文档展示区域
- 完整展示所有支付接口的参数信息
- 区分输入参数和输出参数
- 清晰标注必填项和选填项
- 显示参数类型、示例值和描述
- 采用层级结构布局，便于查阅

## Impact

- Affected specs: payment-test, payment-callback
- Affected code:
  - `web/src/workspaces/PaymentTestWorkspace.tsx` - 新增接口文档展示组件
  - `web/src/lib/paymentApiDocs.ts` - 新增接口文档数据定义

## ADDED Requirements

### Requirement: 接口文档数据定义

系统 SHALL 定义完整的支付接口文档数据结构。

#### Scenario: 定义接口参数
- **WHEN** 定义接口参数时
- **THEN** 包含字段名、变量名、必填标识、类型、示例值、描述

### Requirement: 代收下单接口文档展示

系统 SHALL 展示代收下单接口的完整参数文档。

#### Scenario: 展示请求参数
- **WHEN** 用户查看代收下单接口文档
- **THEN** 显示所有请求参数（商户号、时间戳、签名、通道编码、商户订单号、订单金额、异步通知地址、跳转通知地址、付款人姓名、付款人IP、扩展参数）
- **AND** 标注每个参数是否必填
- **AND** 显示参数类型和描述

#### Scenario: 展示响应参数
- **WHEN** 用户查看代收下单接口响应
- **THEN** 显示所有响应参数（响应状态码、响应信息、返回数据、traceId）
- **AND** 显示data中的具体字段

### Requirement: 代收订单查询接口文档展示

系统 SHALL 展示代收订单查询接口的完整参数文档。

#### Scenario: 展示请求和响应参数
- **WHEN** 用户查看代收订单查询接口文档
- **THEN** 显示完整的请求参数和响应参数

### Requirement: 代付下单接口文档展示

系统 SHALL 展示代付下单接口的完整参数文档。

#### Scenario: 展示请求和响应参数

- **THEN** 显示完整的请求参数和响应参数

### Requirement: 代付订单查询接口文档展示

系统 SHALL 展示代付订单查询接口的完整参数文档。

#### Scenario: 展示请求和响应参数

### Requirement: 商户信息查询接口文档展示

系统 SHALL 展示商户信息查询接口的完整参数文档。

#### Scenario: 展示请求和响应参数
- **WHEN** 用户查看商户信息查询接口文档
- **THEN** 显示完整的请求参数和响应参数

### Requirement: 页面布局设计

系统 SHALL 提供清晰的页面布局展示接口文档。

#### Scenario: 层级结构展示
- **WHEN** 展示接口文档时
- **THEN** 使用可折叠的面板区分不同接口
- **AND** 使用表格形式展示参数信息
- **AND** 必填参数使用特殊标识（如红色星号）

## Interface Documentation Structure

### 通用请求参数
| 字段名 | 变量名 | 必填 | 类型 | 示例值 | 描述 |
|--------|--------|------|------|--------|------|
| 商户号 | merchantNo | 是 | String(128) | 4760774446035387265 | 商户号 |
| 请求时间戳 | timestamp | 是 | long(13) | 1749710334000 | 请求时间戳 |
| 签名 | sign | 是 | String(128) | C380BEC2BFD727A4B6845133519F3AD6 | 签名值 |

### 通用响应参数
| 字段名 | 变量名 | 必填 | 类型 | 示例值 | 描述 |
|--------|--------|------|------|--------|------|
| 响应状态码 | code | 否 | string | 00000 | 00000为成功，其他-处理有误 |
| 响应信息 | message | 否 | String(128) | 处理成功! | 响应的结果信息 |
| 返回数据 | data | 否 | object | {} | 返回响应数据 |
| traceId | traceId | 否 | string | - | 日志排查 |

### 订单状态映射
| 状态码 | 描述 |
|--------|------|
| 0 | 已生成 |
| 1 | 待接单 |
| 2 | 已接单 |
| 3 | 已成功 |
| 4 | 已取消 |
| 5 | 已超时 |
| 6 | 高危订单 |
| 7 | 渠道订单失败 |
| 8 | 匹配失败 |
