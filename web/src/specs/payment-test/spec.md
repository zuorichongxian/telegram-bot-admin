## 技术实现

### 签名算法

根据 API 文档，签名算法步骤如下：

1. 将所有非空参数按参数名 ASCII 码从小到大排序（字典序）
2. 使用 URL 键值对格式拼接：`key1=value1&key2=value2...`
3. 在最后拼接 `&key=` + 商户密钥
4. 对整个字符串进行 MD5 运算
5. 将结果转换为大写

### 页面功能模块

#### 1. 配置面板
- 商户号配置（可编辑）
- 商户密钥配置（可编辑，密码形式显示）
- 通道编码配置（可编辑）
- 回调地址配置（可编辑）
- 基本信息展示区域

#### 2. 代收下单模块
- 输入字段：
  - 订单金额（必填，精确到小数点后2位）
  - 商户订单号（可选，自动生成或手动输入）
  - 付款人姓名（可选）
  - 付款人 IP（可选）
  - 扩展参数（可选）
  - 异步通知地址（必填）
  - 跳转通知地址（可选）
- 显示生成的签名
- 显示请求 JSON
- 执行下单按钮
- 显示响应结果

#### 3. 代收订单查询模块
- 输入字段：
  - 商户订单号（必填）
- 执行查询按钮
- 显示订单状态和详情

#### 4. 代付下单模块
- 输入字段：
  - 订单金额（必填）
  - 商户订单号（可选，自动生成）
  - 收款账户信息
  - 异步通知地址
- 执行下单按钮
- 显示响应结果

#### 5. 代付订单查询模块
- 输入字段：
  - 商户订单号（必填）
- 执行查询按钮
- 显示订单状态和详情

#### 6. 商户信息查询模块
- 一键查询按钮
- 显示商户余额、状态等信息

#### 7. 操作日志面板
- 记录所有 API 调用
- 显示时间、接口、请求参数、响应结果
- 支持清空日志

## UI 设计

### 整体布局
- 采用与现有工作区一致的设计风格
- 使用 Card 组件作为各模块容器
- 柔和的圆角边框和阴影
- 响应式布局，支持移动端

### 颜色方案
- 主色调：stone-900（按钮）
- 成功色：emerald 系列
- 错误色：rose 系列
- 警告色：amber 系列
- 背景色：stone 系列

### 交互设计
- 加载状态：使用 LoadingButton 组件
- 错误提示：使用 flash 消息
- 成功提示：使用 flash 消息
- 表单验证：前端校验必填项

## 安全考虑

1. 商户密钥在前端仅用于测试目的，实际生产环境应通过后端代理
2. 所有敏感信息在日志中脱敏显示
3. 密码输入框使用 type="password" 隐藏密钥

## 兼容性

- 保持与现有代码风格一致
- 使用相同的 UI 组件库（@heroui/react）
- 复用现有的工具函数和类型定义
